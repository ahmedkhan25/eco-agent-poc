RAG Pipeline Architecture

The Retrieval-Augmented Generation pipeline enables semantic search across city planning documents. Documents are processed offline, vectorized, and stored in AWS S3 Vectors. At query time, embeddings are generated and matched against the vector index, with results compressed to fit the context window.

Cost Optimization Strategies

Context Compression

GPT-4o compression reduces token usage by ~80%, significantly lowering API costs.

Session Limits

4-query RAG limit per session controls costs while allowing meaningful research.

Tiered Billing

Pay-per-use tier passes costs to heavy users; unlimited tier provides predictable revenue.
